# 稳稳开店助手-工具中心页面产品说明文档

## 一、页面核心定位与目标

### 1. 定位

作为首页核心诊断功能的延伸，提供**轻量化、即用即走、数据驱动**的开店全周期实用工具，解决创业者"不会算账、不懂避坑、没有数据参考"的核心痛点，是提升用户留存、复访率和PRO版转化的关键页面。

### 2. 核心目标

- 工具使用率≥40%（从首页进入工具中心的用户占比）

- 单工具平均使用时长≥2分钟

- 工具页到PRO版转化率≥8%

- 7日复访率≥20%

### 3. 整体UI规范

完全延续首页设计风格，保持视觉统一：

- 背景：复用首页纹理背景图

- 卡片：白色圆角32rpx，轻微悬浮阴影，内边距36rpx

- 标签：圆角999rpx，不同类型工具使用对应颜色标签（核心工具-蓝色、实用指南-绿色、数据参考-橙色、新功能-红色）

- 图标：统一使用线性+渐变背景图标，尺寸80rpx×80rpx

- 文字层级：工具标题34rpx 700字重，描述26rpx 400字重，标签22rpx 600字重

## 二、五大工具完整设计方案

### 1. 利润测算器（核心工具，优先级最高）

#### UI展示

- 卡片最顶部展示，标注蓝色"核心工具"标签

- 图标：彩色柱状图图标，蓝色渐变背景

- 标题：利润测算器

- 描述：核算固定成本、毛利和盈亏平衡点，算清楚再投钱

- 点击卡片进入独立测算页

#### 核心功能

- 基础版（免费）：

    1. 输入项：房租、水电、人工、装修摊销、设备折旧、原材料成本、客单价、日均客流

    2. 输出项：月固定成本、单客毛利、月盈亏平衡点客流、月盈亏平衡点营收、预计月利润、预计回本周期

    3. 自动生成盈亏平衡分析图

- PRO版（解锁后）：

    1. 增加外卖抽成、营销费用、税费等细分成本项

    2. 支持多场景测算（乐观/中性/悲观）

    3. 生成完整利润分析报告，可保存分享

#### 功能逻辑

1. 用户进入测算页，自动带入之前诊断时填写的行业和阶段信息，预填部分默认值

2. 用户输入数据后，实时计算并更新结果

3. 点击"保存测算结果"，自动同步到用户经营档案

4. 基础版结果页底部固定展示"查看完整利润分析报告（PRO）"转化入口

#### 引流入口

- 首页诊断结果页："立即测算你的利润"按钮

- 经营档案页："我的测算记录"入口

- 分享裂变：生成测算结果海报，分享给好友

#### 数据存储（全部走云数据库）

- 云数据库集合：`profit_calculations`

- 核心字段：`_id`、`openid`（用户唯一标识）、`industryId`、`stageKey`、`input_data`（输入的所有参数）、`result_data`（计算结果）、`created_at`、`updated_at`

- 权限配置：仅创建者可读写

### 2. 开店避坑指南（实用工具）

#### UI展示

- 第二个卡片，标注绿色"实用指南"标签

- 图标：盾牌图标，绿色渐变背景

- 标题：开店避坑指南

- 描述：按行业和阶段整理高频踩坑点，提前知道坑在哪

- 点击卡片进入分类列表页

#### 核心功能

- 基础版（免费）：

    1. 按行业（餐饮/零售/美业等）和阶段（筹备期/开业期/经营期）分类展示坑点

    2. 每个坑点包含：坑点描述、踩坑后果、避坑建议

    3. 支持搜索和收藏坑点

- PRO版（解锁后）：

    1. 查看完整行业避坑手册（每个行业≥50个坑点）

    2. 下载可打印的避坑清单

    3. 专属避坑顾问在线答疑

#### 功能逻辑

1. 进入列表页，默认显示用户已选择行业的筹备期坑点

2. 用户可切换行业和阶段筛选内容

3. 点击单个坑点进入详情页，详情页底部展示"查看完整避坑手册（PRO）"入口

4. 收藏的坑点同步到经营档案的"我的收藏"

#### 引流入口

- 首页诊断结果页：对应风险点直接链接到相关避坑指南

- 下拉刷新好运签：Tips内容链接到对应避坑指南

- 经营档案页："我的收藏"入口

#### 数据存储（全部走云数据库）

- 云数据库集合：`avoid_pit_articles`

- 核心字段：`_id`、`industryId`、`stageKey`、`title`、`content`、`level`（风险等级）、`is_pro`（是否PRO专属）、`created_at`

- 权限配置：所有用户可读，仅管理员可写

- 辅助集合：`user_favorites`，存储用户收藏的坑点ID

### 3. 存活率参考（数据工具）

#### UI展示

- 第三个卡片，标注橙色"数据参考"标签

- 图标：折线图图标，橙色渐变背景

- 标题：存活率参考

- 描述：同行业同城市的店铺存活率数据，心里有个底

- 点击卡片进入数据查询页

#### 核心功能

- 基础版（免费）：

    1. 选择城市和行业，查看该城市该行业近3年的店铺存活率

    2. 展示存活率趋势图和核心数据解读

    3. 对比全国平均存活率

- PRO版（解锁后）：

    1. 查看同商圈精准存活率数据

    2. 查看不同面积、不同投资规模的存活率

    3. 生成专属存活率分析报告

#### 功能逻辑

1. 进入查询页，自动定位用户当前城市，默认显示用户已选择行业的存活率

2. 用户可手动切换城市和行业

3. 数据加载时显示骨架屏，加载完成后展示图表和解读

4. 基础版结果页底部展示"查看同商圈精准数据（PRO）"入口

#### 引流入口

- 首页诊断结果页："查看你所在城市的行业存活率"按钮

- 利润测算器结果页："参考同行业存活率"链接

- 分享裂变：生成存活率数据海报

#### 数据存储（全部走云数据库）

- 云数据库集合：`survival_rate_data`

- 核心字段：`_id`、`city`、`industryId`、`year`、`survival_rate_1y`、`survival_rate_2y`、`survival_rate_3y`、`interpretation`、`is_pro`

- 权限配置：所有用户可读，仅管理员可写

- 注意：数据定期更新，由后台批量导入，不支持用户修改

### 4. 现金流模拟器（新增核心工具，填补空白）

#### UI展示

- 第四个卡片，标注红色"新功能"标签

- 图标：现金流图标，紫色渐变背景

4. 基础版结果页底部展示"生成完整现金流压力测试报告（PRO）"入口

#### 引流入口

- 首页诊断结果页：高风险用户强制展示"立即模拟你的现金流"按钮

- 利润测算器结果页："模拟现金流走势"链接

## 三、后端代码配置逻辑（基于微信云开发）

### 1. 新增云函数清单

|云函数名称|功能描述|入参|出参|
|---|---|---|---|
|`getToolConfig`|获取工具中心所有工具的配置信息|无|工具列表数组|
|`saveProfitCalculation`|保存利润测算结果|openid、industryId、input_data、result_data|保存成功的记录ID|
|`getProfitHistory`|获取用户的利润测算历史记录|openid|历史记录数组|
|`getAvoidPitList`|获取避坑指南列表|industryId、stageKey、page、size|坑点列表、总条数|
|`getAvoidPitDetail`|获取单个坑点详情|id|坑点详情|
|`getSurvivalRate`|获取存活率数据|city、industryId|存活率数据、趋势图数据|
|`saveCashFlowSimulation`|保存现金流模拟结果|openid、industryId、input_data、result_data|保存成功的记录ID|
|`subscribeTool`|预约工具上线提醒|openid、toolId、phone|预约成功状态|
### 2. 云函数通用逻辑

1. 所有云函数入口处校验用户登录状态，未登录用户返回错误提示

2. 所有写操作必须通过云函数执行，禁止前端直接操作数据库

3. 所有查询操作增加分页和缓存，提升响应速度

4. 云函数日志中记录用户操作和错误信息，方便排查问题

### 3. 数据库权限配置

- 所有用户数据集合（`profit_calculations`、`cash_flow_simulations`、`user_favorites`、`tool_subscriptions`）：仅创建者可读写

- 所有公共数据集合（`tool_config`、`avoid_pit_articles`、`survival_rate_data`）：所有用户可读，仅管理员可写

- 管理员权限通过云函数中的`wx.cloud.getWXContext()`获取用户openid，与预设的管理员openid列表比对实现

### 4. 与现有系统对接逻辑

1. 用户体系：复用现有微信授权登录体系，使用`openid`作为用户唯一标识

2. PRO版解锁：复用现有`unlockPro`云函数，所有PRO专属功能在云函数中校验用户的PRO解锁状态

3. 经营档案：所有工具的使用记录和保存结果自动同步到`getArchive`云函数，在经营档案页统一展示

4. 消息推送：工具上线提醒使用微信服务通知，通过云调用`subscribeMessage.send`接口实现

## 四、全页面引流转化逻辑

1. **工具使用前转化**：在工具入口处标注"PRO版解锁更多功能"，吸引用户提前了解PRO权益

2. **工具使用中转化**：当用户使用基础版功能遇到数据限制时，即时弹出PRO版升级提示

3. **工具使用后转化**：工具结果页底部固定展示PRO版升级入口，突出展示PRO版的额外价值

4. **复访引导转化**：用户保存工具结果后，提示"7天后再来测算，看看你的经营状况变化"

5. **裂变转化**：所有工具结果都支持生成分享海报，分享者和被分享者都可获得PRO版试用权益
