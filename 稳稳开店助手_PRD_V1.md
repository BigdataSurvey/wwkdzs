# 稳稳开店助手 PRD（V1.0）

## 1. 文档信息

- 产品名称：稳稳开店助手
- 产品形态：微信小程序
- 文档类型：PRD（产品需求文档）
- 版本号：V1.0
- 适用阶段：MVP 立项 / 原型设计 / UI 设计 / 前后端开发
- 核心目标：围绕“开店前评估值不值得做、开店后判断怎么做得更稳”，打造一款轻量、专业、可复测的创业决策辅助工具。

---

## 2. 产品概述

### 2.1 产品定位

“稳稳开店助手”是一款面向实体门店创业者和轻创业人群的微信小程序，以**开店风险诊断**为核心入口，结合**利润测算、经营建议、AI 场景工具、经营档案沉淀**，帮助用户在开店前降低盲目投入风险，在开店后提升经营决策效率。

### 2.2 产品一句话描述

**不是只告诉你能不能开店，而是告诉你为什么、先改什么、怎么更稳地开。**

### 2.3 产品核心价值

1. 帮用户快速识别开店项目中的关键风险。
2. 将模糊担忧转化为结构化问题拆解。
3. 给出可执行、可排序、可复测的优化建议。
4. 通过经营档案，帮助用户形成持续迭代闭环。

---

## 3. 立项背景与问题定义

基于已有产品说明，当前市场上的同类开店小程序主要以“创业风险测评”为切入口，覆盖行业选择、基础测评、报告生成、辅助工具、AI 能力和广告变现等方向。fileciteturn2file0 但大多数产品存在以下问题：

1. **偏测试化，弱决策化**：能输出结论，但无法形成可执行方案。
2. **偏一次性使用，弱留存**：用户做完测试后缺少继续回来的理由。
3. **内容表达娱乐化**：传播性有余，专业信任感不足。
4. **工具链不完整**：缺少从“判断风险”到“算清账”再到“执行优化”的闭环。

“稳稳开店助手”希望在保留轻量测评入口优势的基础上，建立更专业、更稳定、更利于复访的产品体验。

## 4. 目标用户

### 4.1 核心用户

#### A. 准备开店的新手创业者
- 年龄：22-40 岁
- 状态：有开店想法，尚未正式投入或处于筹备初期
- 典型痛点：不知道该不该做、怕踩坑、怕亏钱、信息不足

#### B. 已开店但经营不稳定的门店主
- 年龄：25-45 岁
- 状态：门店已营业 1-12 个月
- 典型痛点：客流不稳定、利润不清晰、问题很多但抓不住重点

### 4.2 次级用户

#### C. 轻创业人群
- 如美业工作室、零售个体、小吃快餐、咖啡烘焙、社区小店等

#### D. 咨询前置人群
- 想做加盟、想转行、想试水副业的人群

### 4.3 用户分层维度

为提升诊断精度，产品按两个维度切分用户：

1. **行业维度**：餐饮 / 零售 / 美业 / 轻服务业 / 通用开店
2. **阶段维度**：准备开店 / 筹备中 / 已开业 / 想做二店

## 5. 产品目标与衡量指标

### 5.1 业务目标

1. 建立“开店风险诊断”产品认知。
2. 形成“诊断—测算—建议—复测”的产品闭环。
3. 通过激励视频广告与工具使用频次形成初步商业闭环。
4. 验证用户对“经营档案 + AI 助手”的持续需求。

### 5.2 MVP 阶段核心指标

1. 首页进入后发起测评率 ≥ 35%
2. 基础测评完成率 ≥ 65%
3. 报告页到利润测算页转化率 ≥ 20%
4. 次日复访率 ≥ 12%
5. 7 日内二次使用率 ≥ 18%
6. 激励视频广告观看率 ≥ 15%

---

## 6. 产品原则

1. **先专业，再传播**：结论要有依据，文案不过度刺激。
2. **先闭环，再扩功能**：MVP 优先做通核心链路，不堆叠功能。
3. **先行动建议，再抽象评分**：用户看完必须知道下一步怎么做。
4. **先行业适配，再统一模板**：保证结果的针对性。
5. **先低门槛，再做沉淀**：首次使用不强制登录，关键节点再引导授权。

## 9. 核心业务流程

### 9.1 首次使用流程

进入小程序 → 查看首页价值说明 → 点击开始诊断 → 选择行业 → 选择阶段 → 开始基础测评 → 提交 → 查看诊断结果 → 引导进入利润测算 / AI 建议 → 保存到经营档案

### 9.2 二次使用流程

进入小程序 → 档案页查看历史结果 → 选择复测 / 查看未完成建议 → 进入对应工具页执行优化 → 更新记录

### 9.3 变现流程

基础功能免费使用 → 关键节点弹出激励视频 → 解锁深度诊断 / AI 深度建议 / 报告导出等权益

## 10. 页面级 PRD

# 10.1 首页

## 10.1.1 页面目标

让用户在 5-10 秒内理解产品价值，并快速进入核心诊断流程。

## 10.1.2 页面模块

### 模块 A：顶部品牌区
- 展示内容：产品名“稳稳开店助手”、一句话价值主张
- 文案建议：开店前先看风险，开店后更稳经营
- 交互：无

### 模块 B：主行动区
- 主按钮：立即开始诊断
- 次按钮：先算算利润
- 目标：主推测评，辅推利润工具

### 模块 C：快捷入口区
- 基础诊断
- 深度诊断（若未解锁，则展示锁定状态）
- 利润测算
- AI 经营助手

### 模块 D：用户信任区
- 展示内容：
  - 适用行业标签
  - “3分钟快速判断 / 结果可复测 / 提供行动建议”等卖点
  - 风险声明：“结果仅供经营参考，不构成投资建议”

### 模块 E：推荐内容区
- 推荐最近热门问题/常见风险点
- 示例：房租占比过高、客流预估过于乐观、选址判断失真

## 10.1.3 页面规则

1. 首次进入默认展示品牌说明和主按钮。
2. 若用户已有诊断记录，首页顶部增加“继续查看上次结果”按钮。
3. 深度诊断未解锁时显示浅灰锁态，点击触发解锁说明弹窗。

## 10.1.4 埋点建议

- 首页曝光
- 点击开始诊断
- 点击利润测算
- 点击 AI 助手
- 点击继续查看结果

# 10.3 基础风险诊断页

## 10.3.1 页面目标

在较短时间内完成一轮可信的开店风险筛查。

## 10.3.2 功能说明

- 题量：15题
- 答题时长：约 3-5 分钟
- 题型：单选题为主，少量区间题/判断题
- 支持：上一题 / 下一题 / 进度条 / 中断续答

## 10.3.3 题目维度

1. 选址与客流判断
2. 房租与固定成本压力
3. 产品/服务竞争力
4. 客群匹配度
5. 启动资金与现金流安全
6. 回本预期合理性
7. 经营经验与执行力
8. 获客能力与推广计划

## 10.3.4 页面规则

1. 每题必须作答，才可进入下一题。
2. 用户退出时自动保存当前进度。
3. 再次进入时，优先提示“继续上次诊断”。
4. 最后一题提交前展示确认弹窗。

## 10.3.5 输出逻辑

系统根据题目权重计算：
- 总体风险等级
- 各维度得分
- 关键红线项
- 优先改善项

## 10.3.6 埋点建议

- 开始答题
- 每 5 题节点完成
- 中途退出
- 提交完成
- 平均答题时长

# 10.4 诊断结果页

## 10.4.1 页面目标

用专业、清晰、可执行的方式告诉用户：当前项目处于什么水平、核心问题在哪里、下一步该怎么做。

## 10.4.2 结果页结构

### 模块 A：总体结论卡
- 风险等级：低 / 中 / 高
- 结论标题：
  - 低风险：可继续推进，但仍需重点验证
  - 中风险：建议小步试错，优先修正关键问题
  - 高风险：建议暂缓投入，先补齐关键条件
- 补充说明：该结果基于你的当前回答，仅供经营参考

### 模块 B：四维诊断结果
- 资金安全
- 选址与客流
- 产品竞争力
- 经营准备度

展示形式：雷达图或四格评分卡

### 模块 C：关键问题拆解
- 列出 3-5 个核心风险点
- 每个风险点包含：
  - 问题名称
  - 问题说明
  - 影响后果
  - 优先级（高/中/低）

### 模块 D：7 天行动建议
- 建议 1：先做什么
- 建议 2：再验证什么
- 建议 3：哪些钱先不要投

### 模块 E：下一步工具推荐
- 去做利润测算
- 解锁深度诊断
- 让 AI 帮我出优化建议

### 模块 F：经营档案保存
- 引导保存本次记录
- 保存后可查看历史复测变化

## 10.4.3 页面规则

1. 未授权时允许查看结果，但点击“保存档案”时引导微信授权。
2. 点击“深度诊断”时，先触发激励视频广告；观看完成后解锁。
3. 结果页支持分享卡片，但不直接展示过度刺激性标签。

## 10.4.4 文案原则

避免使用：
- 韭菜
- 必死
- 完蛋了
- 不适合创业

建议使用：
- 暂缓投入
- 先验证后扩张
- 风险较高，建议补齐条件
- 当前更适合试营业/小规模试错

## 10.4.5 埋点建议

- 结果页曝光
- 点击利润测算
- 点击深度诊断
- 点击 AI 建议
- 点击保存档案
- 分享结果

# 10.6 利润测算器

## 10.6.1 页面目标

帮助用户把“感觉能赚钱”变成“数字上算得清”。

## 10.6.2 输入项

### 固定成本
- 月租金
- 水电杂费
- 人工成本
- 设备折旧/摊销
- 其他固定支出

### 收入预估
- 客单价
- 日均客流
- 月营业天数

### 毛利相关
- 单单毛利率或原材料成本占比

## 10.6.3 输出项

1. 月营业额预估
2. 月毛利润预估
3. 月净利润预估
4. 盈亏平衡点
5. 回本周期预估
6. 风险提示（如房租占比过高）

## 10.6.4 页面规则

1. 必填关键字段：房租、人工、客单价、客流、营业天数。
2. 缺少非关键字段时允许用默认值估算，但需明确“估算口径”。
3. 结果页支持保存到经营档案。

## 10.6.5 埋点建议

- 打开利润测算页
- 完成输入
- 查看结果
- 保存测算结果

### 11.2 广告规则

1. Banner 广告：放置于首页下方、结果页中下部、工具页底部。
2. 激励视频广告：用于解锁深度诊断、AI 高级建议、报告扩展内容。
3. 不在答题中途强打断，不影响主流程连续性。

### 11.3 分享规则

1. 结果页支持分享卡片。
2. 分享内容强调“风险等级 + 建议方向”，不展示贬损性标签。
3. 后续可配置“邀请好友解锁深度诊断”。

### 11.4 数据保存规则

1. 基础测评结果保留核心结论与时间。
2. 利润测算保存关键参数与结果快照。
3. AI 输出仅保存最近若干条，避免存储压力过大。

## 13. 非功能需求

### 13.1 性能要求

1. 首页首屏加载时间 ≤ 2 秒
2. 题目切换响应时间 ≤ 300ms
3. 结果页生成时间 ≤ 2 秒

### 13.2 合规要求

1. 所有诊断结果仅供参考，不构成投资建议或经营担保。
2. 不得承诺收益、保本、回本期限必达。
3. 用户数据使用需符合微信小程序隐私规范。

### 13.3 可用性要求

1. 页面文案清晰，尽量避免抽象术语。
2. 操作步骤简短，单页面不堆叠过多动作。
3. 异常退出可恢复进度。

## 14. 风险点与应对方案

### 风险 1：测评结果被认为不专业
- 应对：题库围绕真实经营变量设计，减少情绪化标签

### 风险 2：完成率不高
- 应对：首版控制在 15 题内，进度反馈清楚，结果页足够有价值

### 风险 3：留存不足
- 应对：加入经营档案、复测建议、利润测算和 AI 工具承接

### 风险 4：功能过多导致开发周期拉长
- 应对：严格控制 MVP，先做基础诊断 + 结果页 + 利润工具

## 16. 附：首页与结果页建议文案

### 首页主标题建议
- 开店前先查风险，开店后更稳经营
- 3分钟判断项目值不值得做
- 不是拍脑袋开店，而是先把账和风险看清

### 结果页标题建议
- 你的项目可以推进，但先别急着重投入
- 当前风险偏高，建议先修正关键问题
- 这门生意不是不能做，而是要换个更稳的做法

## 17. 本 PRD 的核心结论

“稳稳开店助手”首版不应做成泛化创业百科，也不应仅仅做成情绪化测评工具，而应围绕以下闭环搭建：

**诊断风险 → 拆解问题 → 测算利润 → 给出建议 → 保存档案 → 复测优化**

只有形成这个闭环，产品才既有拉新入口，也有持续留存的抓手，并能逐步建立与同类产品的差异化壁垒。
